Step 1: Create a Dataset

Understanding Datasets

A dataset is a container for tables, views, and models in BigQuery. All datasets belong to a specific Google Cloud project.

Important: The dataset location determines where your data is stored and cannot be changed after creation. Choose wisely based on your users' locations and compliance requirements.

Method 1: Using BigQuery Web UI
  1. Open the BigQuery console
  2. In the Explorer panel, click the three dots next to your project name
  3. Select "Create dataset"
  4. Fill in the details:
    • Dataset ID: my_learning_dataset
    • Data location: us (or your preferred region)
    • Default table expiration: 30 days (optional)
    • Description: "Dataset for learning BigQuery"
  5. Click "Create dataset"
Method 2: Using SQL Command

You can also create a dataset using SQL in the BigQuery query editor:

-- Create a dataset with specific options
CREATE SCHEMA IF NOT EXISTS `my-project.my_learning_dataset`
OPTIONS(
  location = "us",
  default_table_expiration_days = 30,
  description = "Dataset for learning BigQuery"
);

Replace my-project with your actual project ID.

Method 3: Using bq Command Line

If you have Google Cloud SDK installed, use the bq command:

# Create dataset with bq command
bq mk --dataset \
  --location=US \
  --default_table_expiration=3600 \
  --description "Dataset for learning BigQuery" \
my-project:my_learning_dataset

Step 2: Create Table and Insert Data

Create Employees Table

Let's create a simple employees table with the following schema:

Column Name Data Type Description
employee_id INT64 Primary key, unique identifier
first_name STRING Employee's first name
last_name STRING Employee's last name
department STRING Department name
salary NUMERIC Annual salary
hire_date DATE Date when employee was hired
Create Table using SQL

Run this SQL command in BigQuery query editor:

-- Create employees table
CREATE TABLE `my-project.my_learning_dataset.employees` (
  employee_id INT64 NOT NULL,
  first_name STRING,
  last_name STRING,
  department STRING,
  salary NUMERIC,
  hire_date DATE
);
Insert Data into Employees Table

Now let's insert sample data into our employees table:

-- Insert multiple rows into employees table
INSERT INTO `my-project.my_learning_dataset.employees`

  (3, 'Mike', 'Johnson', 'Sales', 70000.00, '2021-01-10'),
  (4, 'Sarah', 'Williams', 'Engineering', 80000.00, '2018-11-05'),

Verify the Data

Run a SELECT query to verify your data was inserted correctly:

Step 3: Create Table from CSV File

Prepare CSV File

First, create a CSV file with product data. Save it as products.csv:

product_id,product_name,category,price,stock_quantity,created_date
101,"Laptop","Electronics",999.99,50,"2023-01-15"
102,"Smartphone","Electronics",699.99,100,"2023-02-20"
103,"Desk Chair","Furniture",199.99,30,"2023-03-10"
104,"Coffee Mug","Kitchen",12.99,200,"2023-01-25"
105,"Notebook","Office Supplies",5.99,150,"2023-04-05"
Upload CSV to Google Cloud Storage
  1. Go to Google Cloud Storage
  2. Create a new bucket or use an existing one
  3. Upload your products.csv file to the bucket
  4. Note the file path: gs://your-bucket-name/products.csv

Tip: If you don't have a bucket, create one with a unique name. Bucket names must be globally unique across all Google Cloud users.

Create Table from CSV using Web UI
  1. In BigQuery console, select your dataset
  2. Click "Create table"
  3. Choose source: "Google Cloud Storage"
  4. Enter the file path: gs://your-bucket-name/products.csv
  5. File format: "CSV"
  6. Table name: products
  7. Check "Auto detect" schema
  8. Advanced options:
    • Header rows to skip: 1
    • Field delimiter: Comma
  9. Click "Create table"
Create Table from CSV using SQL

Alternatively, use SQL to create the table from CSV:

-- Create table from CSV file in Cloud Storage
CREATE TABLE `my-project.my_learning_dataset.products`
OPTIONS(
  description = "Products imported from CSV"
)
AS
SELECT *
FROM `my-project.my_learning_dataset`.products_csv;

Or use the LOAD DATA command:

-- Load data directly from CSV
LOAD DATA INTO `my-project.my_learning_dataset.products`
FROM FILES (
  format = 'CSV',
  uris = ['gs://your-bucket-name/products.csv'],
  skip_leading_rows = 1
);
Verify Imported Data

Check if the data was imported correctly:

-- Verify products data
SELECT *
FROM `my-project.my_learning_dataset.products`
ORDER BY product_id;

Step 4: Create Table from MySQL (Local Machine)

Overview of MySQL to BigQuery

There are several ways to transfer data from MySQL to BigQuery. We'll cover the most practical methods:

Method 1: CSV Export/Import

Export MySQL data to CSV, then import to BigQuery

Method 2: Using Dataflow

Use Google Cloud Dataflow for continuous data transfer

Method 3: Federated Query

Query MySQL directly from BigQuery (limited use cases)

Method 1: CSV Export/Import (Recommended)

This is the simplest method for one-time or periodic transfers:

Step 4.1: Export MySQL Data to CSV

From your MySQL command line or MySQL Workbench:

-- Export query results to CSV
SELECT * FROM your_mysql_database.customers
INTO OUTFILE '/tmp/customers.csv'
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n';

If you can't use INTO OUTFILE, use a tool like MySQL Workbench or phpMyAdmin to export as CSV.

Step 4.2: Create Sample MySQL Data

If you don't have MySQL data, here's a sample table structure and data:

-- Create customers table in MySQL
CREATE TABLE customers (
  customer_id INT PRIMARY KEY,
  customer_name VARCHAR(100),
  email VARCHAR(100),
  city VARCHAR(50),
  country VARCHAR(50),
  registration_date DATE
);

-- Insert sample data
INSERT INTO customers VALUES
  (1, 'Alice Johnson', 'alice@email.com', 'New York', 'USA', '2022-05-15'),
  (2, 'Bob Smith', 'bob@email.com', 'London', 'UK', '2022-06-20'),
  (3, 'Carol Davis', 'carol@email.com', 'Toronto', 'Canada', '2022-07-10'),
  (4, 'David Wilson', 'david@email.com', 'Sydney', 'Australia', '2022-08-05'),
  (5, 'Eva Brown', 'eva@email.com', 'Berlin', 'Germany', '2022-09-12');
Step 4.3: Upload CSV to Cloud Storage
  1. Export your MySQL data to a CSV file (e.g., customers.csv)
  2. Upload the CSV file to Google Cloud Storage
  3. Note the file path: gs://your-bucket-name/customers.csv
Step 4.4: Create Table in BigQuery

Create the customers table in BigQuery from the CSV file:

-- Create customers table from MySQL CSV export
CREATE TABLE `my-project.my_learning_dataset.customers`
(
  customer_id INT64,
  customer_name STRING,
  email STRING,
  city STRING,
  country STRING,
  registration_date DATE
);

-- Load data from CSV
LOAD DATA INTO `my-project.my_learning_dataset.customers`
FROM FILES (
  format = 'CSV',
  uris = ['gs://your-bucket-name/customers.csv'],
  skip_leading_rows = 1
);
Method 2: Using Google Cloud Dataflow

For larger datasets or continuous replication, use Dataflow:

  1. Enable the Dataflow API in Google Cloud Console
  2. Use the MySQL to BigQuery template in Dataflow
  3. Configure the connection parameters:
    • MySQL host, port, database, username, password
    • BigQuery dataset and table names
  4. Run the Dataflow job

Note: Dataflow requires more setup and may incur costs. Use this for production scenarios with large datasets.

Step 5: Verify All Tables

List All Tables in Dataset

Verify that all three tables were created successfully:

-- List all tables in our dataset
SELECT table_name, creation_time, row_count
FROM `my-project.my_learning_dataset`.INFORMATION_SCHEMA.TABLES
WHERE table_type = 'BASE TABLE'
ORDER BY creation_time;
Query Data from All Tables

Run queries to verify data in each table:

-- Check employees data
SELECT 'Employees' AS table_name, COUNT(*) AS record_count
FROM `my-project.my_learning_dataset.employees`
UNION ALL
-- Check products data
SELECT 'Products', COUNT(*)
FROM `my-project.my_learning_dataset.products`
UNION ALL
-- Check customers data
SELECT 'Customers', COUNT(*)
FROM `my-project.my_learning_dataset.customers`;
Join Data from Multiple Tables

Create a comprehensive query joining data from all tables (if they have relationships):

-- Example: If we had orders connecting customers and products
SELECT
  c.customer_name,
  p.product_name,
  e.first_name AS sales_rep
FROM `my-project.my_learning_dataset.customers` c
CROSS JOIN `my-project.my_learning_dataset.products` p
CROSS JOIN `my-project.my_learning_dataset.employees` e
WHERE e.department = 'Sales'
LIMIT 10;
